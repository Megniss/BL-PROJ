# Plan: Bug Fixes

**Phase:** 1 — Fixes + Backend Foundation
**Requirements:** REQ-005, REQ-006
**Depends on:** none
**Parallel:** yes

## Goal

The browse API returns only Available books (not Swapped), and Messages.vue shows visible error messages when fetches fail instead of silently swallowing errors.

## Context

`BookController::browse()` (line 13) filters `where('status', '!=', 'Pending')` which still returns Swapped books. The fix is a one-line change to `where('status', 'Available')`. The existing `BookTest::test_browse_returns_available_books` does not create any Swapped books, so the bug has no test coverage.

`Messages.vue` has two empty catch blocks: `fetchConversations()` (line 155) and `fetchThread()` (line 187). Both have only a Latvian comment `// klusam ignore kludu`. The component's `data()` has no error state properties — `convosError` and `threadError` need to be added.

The `bookloop` SQLite file in the project root is tracked by git but is a leftover artifact. It should be added to `.gitignore` and untracked.

## Tasks

### Fix browse status filter (REQ-005)

- [ ] In `app/Http/Controllers/BookController.php` line 13, change `->where('status', '!=', 'Pending')` to `->where('status', 'Available')`
- [ ] In `tests/Feature/BookTest.php`, update `test_browse_returns_available_books`: add `Book::factory()->count(2)->create(['status' => 'Swapped'])` alongside the existing Pending books, and add an assertion that the response count equals 5 (only the Available ones). Also add a dedicated assertion that none of the returned books have status `Swapped`

### Fix Messages.vue silent error catches (REQ-006)

- [ ] In `resources/js/components/Messages.vue` `data()` (around line 119), add two new properties: `convosError: null` and `threadError: null`
- [ ] In `fetchConversations()` (line 155), replace the empty `catch` block with: `catch (err) { this.convosError = err.response?.data?.message || 'Failed to load conversations.' }`. In the `try` block, add `this.convosError = null` before the axios call to clear the error on retry
- [ ] In `fetchThread()` (line 187), replace the empty `catch` block with: `catch (err) { if (!silent) this.threadError = err.response?.data?.message || 'Failed to load messages.' }`. In the `try` block, add `this.threadError = null` before the axios call. The `silent` guard prevents polling errors from replacing the existing thread display
- [ ] In the template, add error display in the conversation list panel (after the `loadingConvos` div, around line 21): `<div v-else-if="convosError" class="convo-empty text-danger">{{ convosError }}</div>`
- [ ] In the template, add error display in the thread panel (after the `loadingThread` state, before the messages list): `<div v-if="threadError" class="text-center text-danger p-3">{{ threadError }}</div>`

### Clean up bookloop artifact

- [ ] Add `bookloop` (no slash, no wildcard) to the end of `.gitignore`
- [ ] Run `git rm --cached bookloop` to untrack the file without deleting it from disk

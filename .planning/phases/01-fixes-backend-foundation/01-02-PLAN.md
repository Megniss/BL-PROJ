# Plan: Owner ID Migration

**Phase:** 1 — Fixes + Backend Foundation
**Requirements:** REQ-007
**Depends on:** none
**Parallel:** yes

## Goal

The `swap_requests` table has an `owner_id` column that captures the original book owner at swap creation time, and all history/swap-count queries use it correctly instead of the broken `orWhereHas('offeredBook', ...)` clause.

## Context

After a swap is accepted, `SwapRequestController::accept()` transfers book ownership by changing `books.user_id`. This makes any query that derives "who was the original owner" from `books.user_id` unreliable after the swap. The broken pattern appears in three places:

1. `ProfileController::show()` lines 16-21 — swap count query
2. `ProfileController::history()` lines 65-69 — history listing
3. `UserController::show()` lines 18-23 — public profile swap count

All three use `orWhereHas('offeredBook', fn($q2) => $q2->where('user_id', $user->id))` which matches the wrong user after ownership transfer.

The fix is to store `owner_id` (the user who owns the wanted book) at swap creation time and query against it directly.

## Tasks

### Add owner_id migration

- [ ] Create migration: `php artisan make:migration add_owner_id_to_swap_requests --table=swap_requests`
- [ ] In the `up()` method: add `$table->unsignedBigInteger('owner_id')->nullable()->after('requester_id')` — nullable because existing rows need backfill
- [ ] Add a raw statement to backfill existing rows: `DB::statement("UPDATE swap_requests SET owner_id = (SELECT user_id FROM books WHERE books.id = swap_requests.wanted_book_id)")` — note: this uses the CURRENT `books.user_id` which is only correct for pending/declined swaps. For already-accepted swaps the owner has changed, but since the `requester_id` now owns the wanted book, the original owner is the OTHER participant. For accepted swaps, backfill with: `DB::statement("UPDATE swap_requests SET owner_id = (SELECT user_id FROM books WHERE books.id = swap_requests.offered_book_id) WHERE status = 'accepted'")` — after accept, `offered_book.user_id` is the original owner (they received the offered book)
- [ ] After backfill, alter the column to non-nullable: `$table->unsignedBigInteger('owner_id')->nullable(false)->change()`
- [ ] Add foreign key: `$table->foreign('owner_id')->references('id')->on('users')->cascadeOnDelete()`
- [ ] In the `down()` method: `$table->dropForeign(['owner_id']); $table->dropColumn('owner_id');`
- [ ] Run `php artisan migrate` to apply

### Update SwapRequest model

- [ ] In `app/Models/SwapRequest.php`, add `'owner_id'` to the `#[Fillable([...])]` attribute (becomes `#[Fillable(['requester_id', 'owner_id', 'offered_book_id', 'wanted_book_id', 'status', 'requester_dismissed', 'owner_dismissed'])]`)
- [ ] Add an `owner()` relationship method: `public function owner(): BelongsTo { return $this->belongsTo(User::class, 'owner_id'); }`

### Set owner_id in SwapRequestController::store()

- [ ] In `app/Http/Controllers/SwapRequestController.php`, inside the `DB::transaction` closure (line 51), add `'owner_id' => $wantedBook->user_id` to the `SwapRequest::create([...])` array. This captures the owner of the wanted book before any ownership transfer happens

### Fix ProfileController queries

- [ ] In `app/Http/Controllers/ProfileController.php`, `show()` method (lines 16-21): replace the swap count query with `$swapsCount = SwapRequest::where('status', 'accepted')->where(function ($q) use ($user) { $q->where('requester_id', $user->id)->orWhere('owner_id', $user->id); })->count();`
- [ ] In `history()` method (lines 65-69): replace the where clause with the same pattern: `->where(function ($q) use ($user) { $q->where('requester_id', $user->id)->orWhere('owner_id', $user->id); })`

### Fix UserController::show() query

- [ ] In `app/Http/Controllers/UserController.php`, `show()` method (lines 18-23): replace the swap count query with the same pattern: `$swapsCount = SwapRequest::where('status', 'accepted')->where(function ($q) use ($user) { $q->where('requester_id', $user->id)->orWhere('owner_id', $user->id); })->count();`

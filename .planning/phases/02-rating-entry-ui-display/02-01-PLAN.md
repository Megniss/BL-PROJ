Create `resources/js/components/RatingModal.vue` as an Options API component following SwapModal.vue's structure:

**Template structure:**
- `<Teleport to="body">` wrapping `<div v-if="open" class="modal-overlay" @click.self="$emit('close')">` wrapping `<div class="modal-card">`
- Title: `<h2 class="modal-title mb-2">` showing the book title from `book` prop, e.g. `Rate: {{ book?.title }}`
- Star picker: a `<div class="star-picker mb-3">` containing `<span v-for="n in 5" :key="n" class="star" :class="{ active: n <= stars }" @click="stars = n" style="cursor:pointer; font-size:1.5rem">` with Unicode `★` character
- Label above stars: `<label class="form-label fw-semibold">Your rating</label>`
- Review textarea: `<textarea v-model="review" class="form-control mb-3" rows="3" maxlength="500" placeholder="Write a short review..."></textarea>` with a label above it: `<label class="form-label fw-semibold">Review (optional)</label>`
- Error alert: `<div v-if="error" class="alert alert-danger py-2 px-3 mb-3">{{ error }}</div>`
- Button row: `<div class="d-flex justify-content-end gap-2">` with Cancel button (`btn btn-outline-secondary`, emits `close`) and Submit button (`btn btn-primary`, disabled when `stars === 0 || sending`, emits `submit` with `{ stars, review }`)

**Script section (Options API):**
```javascript
import langMixin from '../langMixin.js'

export default {
  name: 'RatingModal',
  mixins: [langMixin],
  props: {
    open:    { type: Boolean, required: true },
    book:    { type: Object,  default: null },
    sending: { type: Boolean, default: false },
    error:   { type: String,  default: '' },
  },
  emits: ['close', 'submit'],
  data() {
    return {
      stars: 0,
      review: '',
    }
  },
  watch: {
    open(val) {
      if (val) {
        this.stars = 0
        this.review = ''
      }
    }
  },
  methods: {
    handleSubmit() {
      if (this.stars === 0 || this.sending) return
      this.$emit('submit', { stars: this.stars, review: this.review })
    }
  }
}
```

**Star styling:** Add a minimal `<style>` block (not scoped) for the star picker:

**Key decisions:**
- Stars and review are internal `data()` (not props) — simpler since parent just reads them on submit event (per research recommendation)
- `watch: { open }` resets stars/review to 0/'' when modal opens — prevents stale state
- Submit emits `{ stars, review }` payload so parent can read values without v-model wiring
- `maxlength="500"` on textarea enforces REQ-002 frontend limit
- No `langMixin` usage in template text for now — hardcoded English strings. Translation keys will be wired in plan 02-03 when translations.js is updated. This keeps plan 01 self-contained.
